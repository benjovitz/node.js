Git in the Terminal

Working with git is essential for a developer for accessing and sharing code. Working with it in the terminal is the rawest form which doesn't make you depended of a program or extension to push or pull your code.

Clone an existing repository

Pushing to a repository

We now have a repository with where to push is already defined in its .git folder. So pushing to it is easy!
git add "filename" to add a specific file to your staging area or git add -A to add all changes.
git commit -m "message here" to create a commit from the staging area, if no -m (message) is included you will be lead to VI or VIM to add a message.
git push to push to the origin.

git commit -m "commit message"

Pushing to a new repository

You now have a project on your computer but need to push that code to a repository not yet created.
Create the repository on github or other git sites.
git init to initialize a .git folder in your project, which holds all the information on where to push, which branch you are on and so on.
git add -A adding all changes to the staging area like before.
git commit -m "message" creating a commit.
git remote add origin https://github.com/username/newProject adding our origin place for pushing and pulling
git branch -M main renaming default branch to main instead of master.
git push -u origin main first push needs to be like this. The -u is for setting up the upstream to origin (which we specified in the line above, like a variable) and main to specify the branch. 
git init
Initialized empty Git repository in /home/daniel/kode/test/.git/
git add -A
git commit -m "testing"
