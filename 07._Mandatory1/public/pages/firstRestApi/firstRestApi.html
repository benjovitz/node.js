Rest conventions

There are some REST conventions to follow to make sure the is consitency in different REST api's.

Order

The order of the endpoints in the specific app is to make it easy for developers to find a specific endpoint because they know where to look for it. The order is as follows:

get

Which is used to get information from the server. The only info the client can send is via parameters.

post

Which is used to create new instances.

patch

Which is used to update one or more properties of a single instance.

put

Which is used to update all of the properties of a single instance.

delete

Which is used to delete a single instance.

Naming

Lets say we have a database of octopuses. The endpoint name would reflect what entity we are affecting and it should be plural.

get: /octopuses

post: /octopuses

patch: /octopuses/:id

put: /octopuses/:id

delete: /octopuses/:id

nodemon

Nodemon is a tool that will listen to changes in specific file extensions and restart the server on those changes. What file extensions nodemon should listen to can be set in a nodemon.json file. 
nodemon.json
{
    "ext" : "html,js,json"
}

Scripts

We can setup up custom scripts to run commands via npm, i have added the start:script script.
To run it, we will need to write npm run start:script.
Scripts which a named start, test and build doesnt need run before it. The reason we need run before it, is because we can name them anything. So we could name a script install which conflicts withnpm install.
I have also added type: module. The default is type: commonjs. There is a difference in how to import with the two types. See examples below. 
package.json
{
    "name": "node",
    "version": "1.0.0",
    "description": "",
    "main": "index.js",
    "type": "module",
    "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
        "start:script": "node ./app.js"
    },
    "keywords": [],
    "author": "",
    "license": "ISC"
}

export.js
    const exportThis = "EXPORT ME!"

    //commonjs export
    module.exports = {
        exportThis
    }
    //module export
    export { exportThis }


import.js
    //commonjs import
    const importedValue = require("./export.js")

    //module import
    import {exportThis} from "./export.js"
